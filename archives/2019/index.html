接口 https://typescript.bootcss.com/interfaces.html 在TypeScript里，接口的作用就是定义结构。 可选属性名字( ? ), 可以对可能存在的属性进行预定义，以及捕获引用了不存在的属性时的错误。 可以将接口属性定义为 readonly，定义后只能在初始化时接口对象时赋值。 ReadonlyArray，只读数组，定义后不可更改，同时也不可将其复制给普通数组。 变量使用const，属性使用readonly。 对象字面量会被特殊对待且会额外属性检查，当将其赋值给变量或作为参数传递的时候。 如果一个对象字面量存在任何“目标类型”不包含的属性时，会报错，可以用以下方法规避： 类型断言（使用类型断言可以绕开类型检查） 字符串索引签名（eg: man[‘canRead’]） 将该对象赋值给一个另一个变量 接口表示函数类型，需给接口定义一个调用签名。 interface SearchFunc { (source: string, subString: string): boolean; } let mySearch: SearchFunc; mySearch = function(source: string, subString: string) { let result = source.search(subString); return result > -1; } 可索引的类型，具有一个索引签名，它描述了对象索引的类型，还有相应的索引返回值类型。 可以同时使用 string 和 number 两种类型的索引，但是数字索引的返回值必须是字符串索引返回值类型的子类型。（使用 number 类型索引时会自动将其转化为 string 类型的索引） 字符串索引签名能够很好的描述 dictionary 模式，并且它们也会确保所有属性与其返回值类型相匹配 (obj.property 和 obj[“property”]) [key: string]: string | number; // 必须包括所用成员类型 索引签名可设置为只读（readonly） class 强制一个类去符合某种契约 接口描述了类的公共部分，而不是公共和私有两部分。 它不会帮你检查类是否具有某些私有成员。 当你用构造器签名去定义一个接口并试图定义一个类去实现这个接口时会得到一个错误:...

变量声明 https://typescript.bootcss.com/variable-declarations.html var 声明可以在包含它的函数，模块，命名空间或全局作用域内部任何位置被访问。 多次声明同一个变量并不会报错 声明的变量可以在包含它们的函数外访问 let 声明的变量使用的是词法作用域或块作用域。 块作用域变量在包含它们的块或 for 循环之外无法访问 不能在被声明之前读或写(仍可在一个拥有块作用域变量被声明前获取它，只是我们不能在变量声明前去调用此函数) 在同一作用域只能声明一次 每进入一个作用域就会创建一个变量的环境，就算作用域内代码已经执行完毕，这个环境与其捕获的变量依然存在 const 拥有与 let 相同的作用域规则，但不能对它们重新赋值。 变量内部状态可修改(可通过设置对象成员为只读改变) 使用最小特权原则，所有变量除了计划去修改的都应使用const。 基本原则：如果一个变量不需要对它写入，那么其它使用这些代码的人也不能够写入它们，并且要思考为什么会需要对这些变量重新赋值。 使用 const 可更容易推测数据的流动。 解构数组： let input = [1, 2]; let [first, second] = input; [first, second] = [second, first]; // wap variavle function f([first, second]: [number, number]) let [, second, , fourth] = [1, 2, 3, 4]; 可用作在一行内声明多个数组变量，用作自解释 元组结构 let tuple: [number, string, boolean] = [7, "hello", true]; let [a, ....
