TS文档阅读小结1 基础类型

基础类型

  1. https://typescript.bootcss.com/basic-types.html
  2. TypeScript 里的所有数字都是浮点数。
  3. 模版字符串，可以定义多行文本和内嵌表达式。 其被反引号包围（ `），并且以 ${ expr } 这种形式嵌入表达式。
  4. 数组，使用后跟[]元素的类型来表示该元素类型的数组或者使用通用数组类型Array。

  5. 元组类型( Tuple )允许表示一个已知元素数量和类型的数组，各元素的类型不必相同。

    1. eg: let x: [string, number]; x = [‘hello’, 10]; // 必须要按照定义类型的顺序赋值

    2. 访问越界元素会出现错误提示:

      x[3] = "world"; // Error, Property '3' does not exist on type '[string, number]'.
        
console.log(x[5].toString()); // Error, Property '5' does not exist on type '[string, number]'.

  6. 枚举类型提供的一个便利是你可以由枚举的值得到它的名字。

  7. 类型检查器不对 any 类型的值进行检查而是直接让它们通过编译阶段的检查。Object 类型的变量只允许给它赋任意值 ，但是却不能够在它上面调用任意的方法，即便它真的有这些方法。

  8. 指定了 –strictNullChecks 标记(编译选项)，null 和 undefined 只能赋值给 void 和它们各自。

  9. never 类型表示的是那些永不存在的值的类型，即使 any 也不能赋值给 never。

    1. 可作为 error 函数 (return Error()) 的返回值

  10. 类型断言( as , 尖括号 (<String>)) 类似 JAVA 中的类型转换，但是不进行特殊的数据检查和解构。不影响运行，只干预编译阶段。

  11. 推荐使用 let 代替 var（和作用域有关）
