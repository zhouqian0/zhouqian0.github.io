Mysql 必知必会阅读小记1

  1. 数据库(database) 保存有组织的数据的容器(通常是一个文件或一组文件)。
  2. 我们日常使用的是 DBMS(数据库管理系统)，而非直接操作数据库，mysql 就是一种 DBMS。
  3. 关于主键的好习惯:
    • 不更新主键列中的值;
    • 不重用主键列的值;
    • 不在主键列中使用可能会更改的值。
  4. SQL 是结构化查询语言(Structured Query Language) 的缩写。
  5. SHOW DATABASES;
USE XXX;                                    // 选定特定的数据库;
SHOW TABLES;                                // 显示选定数据库的所有表;
SHOW COLUMNS FROM XXX; == DESCRIBE XXX;     // 展示选定数据库表的详情;
SHOW STATUS;                                // 用于显示广泛的服务器状态信息
SHOW CREATE DATABASE XXX;                   // 显示创建特定数据库的MySQL语句;
SHOW CREATE TABLE XXX;                      // 显示创建特定表的MySQL语句;
SHOW GRANTS;                                // 用来显示授予用户(所有用户或特定用户)的安全权限;
SHOW ERRORS;                                // 用来显示服务器错误消息;
SHOW WARNINGS;                              // 用来显示服务器警告消息;
HELP SHOW;                                  // 展示所有可以通过 show 来执行的命令;

  6. SELECT DISTINCT vend_id * FROM products; // 去重查找 vend_id

    DISTINCT 关键字应用于所有列而 不仅是前置它的列。如果给出 SELECT DISTINCT vend_id, prod_price，除非指定的两个列都不同，否则所有行都将被检索出来。

  7. SELECT prod_name * FROM products LIMIT 5;           // 查找前 5 行
SELECT prod_name * FROM products LIMIT 5, 5;        // 从第 6 行开始往后查找 5 行（行数从 0 开始）
SELECT prod_name * FROM products LIMIT 3 OFFSET 5;  // 从第 6 行开始往后查 3 行

  8. SELECT prod_name * FROM products ORDER BY prod_name; // 通过 order by 对检索数据进行排序  
SELECT prod_name FROM products ORDER BY prod_price, prod_name; // 先按照 prod_price 排序，当 price 一样时，按照 prod_name 排序
SELECT prod_name FROM products ORDER BY prod_price DESC LIMIT 1; // 查找价格最贵的一个商品
    • order by 所跟数据可以不是 select 的数据。
    • order by 所跟数据默认为 ASC 升序。
    • ASC、DESC 只对当个列有效，如果需要指定多个列的升降序，需要在每个列前显示指定 ASC/DESC。
  9. SELECT prod_name FROM products WHERE prod_price BETWEEN 5 AND 10;
SELECT prod_name FROM products WHERE prod_price IS NULL;
    • where 和 order by 同时出现时，order by 需要在 where 之后。
    • where 匹配时默认不区分大小写
  10. SELECT prod_name FROM Products WHERE prod_id = 'bnbg01' AND prod_price <= 10;
SELECT prod_name FROM Products WHERE prod_price >= 5 OR prod_price <= 3;
SELECT prod_name FROM Products WHERE prod_price IN (3.49 , 9.49); // 等价于  prod_price = 3.49 or prod_price = 9.49
SELECT prod_name FROM Products WHERE prod_price NOT IN (3.49 , 9.49);
    • 当 and 和 or 同时出现时，如果未用括号进行合理分隔，则会优先计算 and 部分语句。
    • in 操作符一般比 or 操作符清单执行更快。
    • in 可以包含其他 select 语句。
    • 灵活运用 not 对 in、between、exists 子句取反。
