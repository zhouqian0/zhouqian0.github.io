Go 学习小记2

本文将介绍 golang 的 Packages, Switch, Array, Slices, Variadic Functions…

Packages

  1. The order of initialisation of the main package:
    1. The imported packages are first initialised.
    2. Package level variables are initialised next.
    3. init function is called.
    4. main function is called at last.

Switch

  1. A fallthrough statement is used to transfer control to the first statement of the case that is present immediately after the case which has been executed.
    • fallthrough should be the last statement in a case.
    • fallthrough should not appear in the last case.

Array

  1. Array can even ignore the length in the declaration and replace it with … and let the compiler find the length for you.
    • a := [...]int{12, 78, 50}
  2. The size of the array is a part of the type.
    • Hence [5]int and [25]int are distinct types.
  3. Arrays in Go are value types and not reference types.

Slices

  1. Slices is just references to existing arrays.
  2. The capacity of the slice is the number of elements in the underlying array starting from the index from which the slice is created.
  3. veggies := []string{"potatoes", "tomatoes", "brinjal"}
fruits := []string{"oranges", "apples"}
food := append(veggies, fruits...)
  4. When a slice is passed to a function, even though it’s passed by value, the pointer variable will refer to the same underlying array.
  5. Slices hold a reference to the underlying array. As long as the slice is in memory, the array cannot be garbage collected.
    • One way to solve this problem is to use the copy function to make a copy of that slice (original array can be garbage collected.). 
      countries := []string{"USA", "Singapore", "Germany", "India", "Australia"}
neededCountries := countries[:len(countries)-2]
countriesCpy := make([]string, len(neededCountries))
copy(countriesCpy, neededCountries) //copies neededCountries to countriesCpy
return countriesCpy

Variadic Functions

  1. Only the last parameter of a function can be variadic.
  2. The way variadic functions work is by converting the variable number of arguments to a slice of the type of the variadic parameter.
  3. A way to pass a slice to a variadic function:
    • suffix the slice with ellipsis …
  4. append returns a new slice if the capacity is not enough.
