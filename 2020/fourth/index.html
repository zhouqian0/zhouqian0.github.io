gRPC 学习小记

初次接触 gRPC 的学习过程记录。

RPC

  1. 使用 RPC 的优势
    • 长链接，相对于每次建立 HTTP 请求的三次握手，减少网络开销
    • 可以像调用本地的函数一样去调远程函数
  2. 实现 RPC 存在的问题
    • client-server 数据传输通道的建立，大部分是 TCP 连接，gRPC 使用 HTTP2。
    • Call ID映射（在RPC中，所有的函数都具有独立的ID。客户端发起远程过程调用时须带上此ID。）
    • 序列化和反序列化（客户端和服务端的传输的数据）

gRPC

  1. gRPC 提供了一种简单的方法来精确地定义服务和为 iOS、Android 和后台支持服务自动生成可靠性很强的客户端功能库。 客户端充分利用高级流和链接功能，从而有助于节省带宽、降低的 TCP 链接次数、节省 CPU 使用、和延长电池寿命。
  2. 优点
    • 内容交换格式采用 ProtoBuf(Google Protocol Buffers)，使用二进制，（反）序列化速度快，压缩效率高。
    • 传输协议 采用http2。

Protocol Buffers

  1. Protocol Buffers 是 Google 开发的 一种跨语言和平台的序列化数据结构的方式 ，是一个灵活的、高效的用于序列化数据的协议。
  2. 与 XML 和 JSON 格式相比的优势：
    • are simpler
    • are 3 to 10 times smaller
    • are 20 to 100 times faster
    • are less ambiguous
    • generate data access classes that are easier to use programmatically
